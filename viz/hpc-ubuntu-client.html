新 client 接入 Lustre 的 checklist

// role 按设计只装软件 · 网络与挂载每台机显式声明(防误挂生产)

host vars 四件套

  • mlx5_ib_ipoib.enabled=true + interfaces(netplan 65-mlx5-ib.yaml)
  • IB IP 按规律:researchN → 1.1.1.(N+1)/24(MTU 2044)
  • lustre_mgs_nid 指向目标集群 MGS
  • lustre_auto_mount=true 才会写 fstab + 挂载

运维铁律(client 侧)

  • LNET/ko2iblnd 保持默认(peer_credits=32)—— 调高会触发 recovery storm 拖崩 client
  • client 重启/关机前必须先 umount -l Lustre,否则 lnet 卡死 systemd 只能 IPMI 救
  • Ansible 传输:ANSIBLE_SCP_IF_SSH=true + pipelining(sftp 子系统会挂)

已知硬件案例

  • research11 ConnectX-5 fw 16.21.3002:每次重启 irisc 崩一次 = eviction/写卡死根因,治本=换卡
  • 「cannot bind to a free privileged port -99」是连不上 research11 的伪装错误,别冤枉 OSS
  • fsync 卡死 unkillable 进程清理 → sysrq-b(见 memory 手法)
