预编译产物 / Katello 仓

// 全在 foreman pulp,装机不出内网 · vermagic 与 versionlock 成对

lustre-server-modules-2.17.0

  • 28 个 .ko(含 osd_ldiskfs),vermagic 钉 5.14.0-611.55.1.el9_7
  • 仓:Lustre_HPC/Rocky9.7_lustre_server(id 74)
  • 装法:rpm -ivh --replacepkgs → depmod -a,不装 dkms

doca-ofed-modules-26.01

  • 25 个 .ko,Provides 6 个 -dkms 包名 → dnf install doca-ofed 只装用户态、不触发编译
  • 仓:NVIDIA_DOCA/Rocky9.7_doca(id 79)
  • 暴露过的坑:e2fsprogs 要用 dnf 装(解 info 依赖)、验证改查 osd_ldiskfs.ko(预编译没有 dkms)

Rocky_9_BaseOS_Debug(id 73)

  • 仅 DKMS 路径需要:osd-ldiskfs 编译要 kernel-debuginfo(ext4 源)
  • 钉死 el9_7 —— 用 stock 9.8 debuginfo 会导致 ldiskfs 不编(踩过)
  • 预编译路径完全不碰它

LNet / ko2iblnd 与验证清单

// role 模板下发 /etc/lnet.conf + modprobe.d/ko2iblnd.conf

服务端 ko2iblnd 调优(role 下发)

  • credits=4096 · ntx=4096 · concurrent_sends=256 · conns_per_peer=2(ConnectX-5 100G)
  • handler「reload ko2iblnd」只在参数实际变化时智能 rmmod+modprobe
  • ⚠ 铁律:client 侧 LNET/ko2iblnd 保持默认(peer_credits=32),乱调会触发 recovery storm;改前必读 lustre_poc_tuning 手册

jobstats(MGS 专属阶段)

  • lctl set_param -P jobid_var=procname_uid 写 MGS llog,全 OST/MDT 下发
  • 重启不丢(非持久值丢失正是 Grafana jobstats 没数据的老根因)
  • 幂等:非 MGS / 未挂载阶段自动跳过

装完验证清单

  • dkms status → 空(零编译)
  • lsmod | grep osd_ldiskfs → 就位
  • lctl list_nids → 如 1.1.1.38@o2ib
  • ofed_info -s → DOCA 26.01;dnf versionlock list → kernel×4 锁定
