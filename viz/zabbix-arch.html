Zabbix Item / 接口清单

// UserParameter 定义于 zabbix_agentd.d · item 必须 type=7 (agent active) · value_type=text

UserParameter(agent 侧)

  • aidc.hw.snapshot — 读快照 JSON(text · 5m · history 30d)
  • aidc.hw.locate[ref,mode] — 磁盘定位灯 on/off(反向控制)
  • net.if.ip[if] / net.if.ip.list[*] — 网卡 IP 查询(ib0/ib1 等)
  • agent 配置:ServerActive=192.168.6.13 · RefreshActiveChecks=5

AIDC 后端调用(FastAPI)

  • integrations/zabbix.py — JSON-RPC item.get,Authorization: Bearer
  • integrations/hardware.py — 解析 lastvalue JSON → 入库缓存
  • 缓存表 ServerHardwareSnapshot,支持 force_refresh=true 强刷
  • 温度/功耗走 get_host_metrics() 实时 item.get,不落缓存

REST 端点(前端消费)

  • /servers/{id}/hardware — DIMM/NIC/RAID/磁盘全量
  • /servers/{id}/hover-detail — CPU温/进风温/功耗 + 网卡 IP
  • /racks/{id}/detail — U位占用/状态灯/活动告警/PDU
  • 状态灯优先级:Prometheus ipmi_up → Zabbix 兜底

「No hardware snapshot」排障速查

// 完整手册:wiki-docs/infra/aidc-hardware-snapshot-troubleshooting.md · 6 类根因

采集端(机上)

  • 没装:跑 zabbix_client role 补齐(gpu01 案例)
  • 自测:zabbix_agentd -t aidc.hw.snapshot 返回 JSON 才算好
  • SELinux 大坑:快照目录须 zabbix_var_lib_t 标签,拒绝被 dontaudit 静默
  • UserParameter 别带 sudo;Hostname= 必须与 Zabbix host 名一致(pve00 案例)

服务端(Zabbix)

  • item 类型必须 7(agent active);建成 trapper(2) 则永远 never(nfs01 案例)
  • host 被禁用 status=1 → 该机所有 active item 全 never(k8snode03 案例)
  • 排查:item.get 看 type/state/lastclock;同机 agent.ping 也 never 先查 host status

消费端(AIDC)

  • 提示 "No hardware snapshot has been reported" = item lastvalue 为空
  • 硬件 Tab 点 force_refresh 可绕缓存直拉 Zabbix 验证
  • 悬停卡无温度/功耗 = hw.* 数值 item 缺失(独立于快照链路)
