Data Pipeline

Four steps from physical sensors to normalized metrics

01

IPMI Raw Collection

Zabbix IPMI Poller connects to each server BMC via IPMI 2.0 protocol. The ipmi.get master item fetches all sensor data at once, returning a JSON array.

ipmi.get → JSON Array
02

Macro Vendor Mapping

Each server has 4 host-level macros mapping vendor-specific sensor names to unified variables. Supermicro's CPU0_Temp and Dell's Temp point to the same metric.

{$HW_CPU_TEMP} = CPU0_Temp
03

JSONPath Extraction

Dependent items use JSONPath expressions to extract target sensor values from the master item. Discard-on-error ensures missing sensors don't cause failures.

$.[?(@.id=="{$HW_CPU_TEMP}")].value
04

Unified Output

All servers output the same 4 standardized metrics, ready for dashboard visualization, trigger alerting, and cross-vendor comparison.

hw.cpu.temp · hw.inlet.temp · hw.power.total
