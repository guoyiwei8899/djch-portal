Foreman 装机流水线 · Rocky 9.7

Build Host → PXE GRUB2 UEFI → Kickstart(Katello + IPA)→ Ansible Roles。两种启动 Ansible 的方式:build 后原生 roles 自动触发 与 REX 单跑,执行器同为 smart-proxy ansible-runner

hostgroup Rocky 9.7 Server(id 1) PXE Loader Grub2 UEFI 模板 id195 KS / id196 PXEGrub2 自动触发 post_provision 900s REX 用户 turing

两种启动 Ansible 的方式

// 同一个执行器(smart-proxy ansible-runner),不同触发与变量入口
维度① Build → 原生 Roles(自动)② REX 单跑(手动/定时)
触发host built 回调 → Ansible 插件在 ansible_post_provision_timeout=900s 窗口自动调度UI「Run Job」/ hammer job-invocation create / 定时计划
Job 形态名称「Run Ansible roles」· 类别 Ansible Playbook(插件按 hostgroup 绑定的 role 自动生成 playbook)任意模板:Play Ansible roles / Ansible playbook / 命令(实例:Daily non-kernel security 04:00)
Role 来源hostgroup「Rocky 9.7 Server」显式绑定 5 role(meta 依赖不自动跑,mlx5_ib 必须显式)job 里现选 role / playbook / 命令,与 hostgroup 绑定无关
变量来源四层优先级:defaults → Ansible Variables(Override)→ hostgroup 参数 → host 参数同左 + job 级 extra vars(临时覆盖)
典型用途裸机纳管全自动(job#81:全新装机零编译实证)调试重跑(06-08 那串 failed→succeeded)、日常安全更新、ad-hoc

Build 态 menuentry(模板 id196 渲染)

menuentry 'Rocky Linux 9.7 Install' {
  linux boot/Rocky-9.7-x86_64-vmlinuz
    inst.repo=http://192.168.6.15:8080/rocky9-iso/Minimal
    inst.ks=<foreman_url('provision')>
    inst.stage2=http://192.168.6.15:8080/rocky9/ inst.noverifyssl
  initrd boot/Rocky-9.7-x86_64-initrd.img
}
  • TFTP 路径:/var/lib/tftpboot/grub2/grub.cfg-01-<mac>(MAC 冒号转横线)
  • 内核/initrd 同放 tftpboot/boot/(Rocky-9.7 + aliyun 媒介两套)

变量四层优先级(原生 roles 模式)

// 后者覆盖前者 · 选项卡④有动画走向

① role defaults(打底)

  • 磁盘 /etc/ansible/roles → /opt/ansible 软链
  • 例:lustre_prebuilt_rpm_url 默认已指 Katello 预编译仓

② Ansible Variables

  • role 导入 Foreman,勾 Override 才注入 play
  • ⚠ 导入默认值会陈旧(曾押旧 IP/失效 URL,2026-06-12 已全量对账修复 11 个)—— 改磁盘 defaults 后记得同步 Foreman 变量

③ hostgroup 参数

  • ansible_user=turing · ansible_become=true
  • ansible_ssh_private_key_file = foreman-proxy 专用 key

④ host 参数(最高)

  • 每台机个性化:ib_interfaces(yaml 型)定 IB IP
  • 实例:lustre04 → ib0 = 1.1.1.38/24(NID 1.1.1.38@o2ib)
