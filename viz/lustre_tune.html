两个根因

// 一个让服务端连不上,一个让 client 没退路

① 特权端口耗尽 → -99

  • ko2iblnd 发起出向连接默认绑 特权端口(512-1023),仅 ~512 个
  • 长期重连 churn 把池子漏干 → cannot bind to a free privileged port: rc=-99
  • ko2iblnd 连接竞争:NID 大者发起;服务端(.2X) > client(.X+1),故服务端恒为发起方,-99 砸在服务端
  • 结果:服务端建不了任何新出向连接,只剩端口耗尽前的老连接;重启过的 client 全连不回
  • 解:use_privileged_port=0 → 改用 ~28000 个临时端口,永不耗尽

② discovery 不一致 → 无退路

  • OSS/MDS 双口本应互为冗余(ib0=.2X / ib1=.22X)
  • 服务端 discovery=0 → 不对外通告第二个 NID
  • 新重启的 client 只拿到单轨 .2X,主口一坏 没有备口可切
  • 老 client(research10)早建好双轨 MR peer,所以能自动绕 .223 活下来
  • 解:服务端 discovery=1 + client 静态 lnet_mr_peers 兜底(首连主口已死也知备口)

Ansible 落地

// roles/lustre_client + lustre_server · 改 defaults/模板,下次 playbook / reboot 生效(非在线热改)

ko2iblnd.conf(modprobe.d)

# roles/*/...ko2iblnd
options ko2iblnd peer_credits=32 ... fmr_cache=1 \
                 use_privileged_port=0 \
                 require_privileged_port=0
  • 需 rmmod/modprobe ko2iblnd 或 reboot 生效

lnet.conf(global + 多轨 peer)

# roles/lustre_client/templates/lnet.conf.j2
global:
    discovery: 1
    health_sensitivity: 100
    transaction_timeout: 20
    retry_count: 2
peer:
  - primary nid: 1.1.1.23@o2ib
    Multi-Rail: True
    peer ni:
      - nid: 1.1.1.23@o2ib   # ib0
      - nid: 1.1.1.223@o2ib  # ib1 备口

defaults/main.yml(变量)

ko2iblnd_use_privileged_port:     0
ko2iblnd_require_privileged_port:  0
lnet_discovery:           1
lnet_health_sensitivity:  100
lnet_transaction_timeout: 20
lnet_mr_peers:
  - {primary: 1.1.1.21@o2ib, nids:[...221]} # l01
  - {primary: 1.1.1.22@o2ib, nids:[...222]} # l02
  - {primary: 1.1.1.23@o2ib, nids:[...223]} # l03
  • 双口服务器 host_vars 须把 ib0+ib1 都列进 ib_interfaces

应急手册 · 零干扰

// 不改配置、只影响目标这一对、其它 client 毫无感知

A · 服务端主动打通(赢家方向)

# 在坏掉那一口的服务端,用该口源主动连 client
lnetctl ping --source <坏口NID> <clientNID>
# 例:lustre03 ib0 死,救 research2
lnetctl ping --source 1.1.1.23@o2ib 1.1.1.3@o2ib
  • ko2iblnd 竞争里 NID 大者(服务端)才是合法发起方;它主动建起的 conn 双向可用
  • 前提:服务端该口出方向没被 -99 卡死;若卡死改用 B

B · client 手动补多轨(绕坏口)

# 在 client 上给服务器补第二个 NID 成多轨
lnetctl peer add --prim_nid <主NID> --nid <备NID>
# 例:研2 绕 lustre03 ib1(.223)到 OST0003
lnetctl peer add --prim_nid 1.1.1.23@o2ib \
                 --nid 1.1.1.223@o2ib
# → OST0003 立即 FULL,经 .223 读写恢复

排错口诀

  • client 挂不上 + consumer defined fatal error + IPoIB 通 + 别的 client 能连同一 NID → ko2iblnd 连接死锁,先试 A
  • 服务端日志 cannot bind to a free privileged port -99 → 该服务端出方向假死(端口耗尽),A 会失败 → 用 B 绕备口,治本上 use_privileged_port=0 + 重启该服务端 lnet
  • 千万别 lnetctl net del/add MDS 的 ib0 NI / 重载 ko2iblnd —— 全体 client 同时掉 MDC,recovery storm
  • lnetctl peer del(不带 --force)对 Lustre 建的 peer 是空操作;evict/client 重启都治不了这个死锁
