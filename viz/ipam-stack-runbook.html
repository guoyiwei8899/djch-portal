1 phpIPAM 部署

// 自包含 MariaDB · Longhorn · Traefik ingress · SealedSecret

1.1DB 密码 secret(随机,不落明文)

# 在 k8s 节点(sudo kubectl,KUBECONFIG=/etc/rancher/rke2/rke2.yaml)
kubectl -n infra-nt create secret generic phpipam-db \
  --from-literal=mysql-root-password=$(openssl rand -hex 16) \
  --from-literal=mysql-password=$(openssl rand -hex 16)

1.2清单:MariaDB(StatefulSet)+ phpipam-www + Service + Ingress

坑:集群有 两个 default StorageClass(local-netbox + longhorn),PVC 必须显式 storageClassName: longhorn,否则可能错绑 local。
# 关键 env(phpipam-www Deployment)
IPAM_DATABASE_HOST=phpipam-mariadb   IPAM_DATABASE_NAME=phpipam
IPAM_DATABASE_USER=phpipam           IPAM_DATABASE_PASS=<secret>
IPAM_TRUST_X_FORWARDED=true   # 认 Traefik X-Forwarded-Proto=https,否则 302 发 http
IPAM_DISABLE_INSTALLER=true  # schema 手动导入后关向导
# 镜像钉死 phpipam/phpipam-www:v1.8.1(nullconfig helm chart 的 gh-pages 已 404,弃用)
kubectl apply -f manifests/infra-nt/phpipam.yaml

1.3导入 SCHEMA(首启若因 DNS 未就绪跳过了自动导)

症状:web 报 theme on null 或 Could not connect to database=空库。www 首启早于 mariadb ready 时会漏导 schema。
kubectl -n infra-nt exec deploy/phpipam-www -- cat /phpipam/db/SCHEMA.sql \
 | kubectl -n infra-nt exec -i phpipam-mariadb-0 -- mariadb -uphpipam -p<pw> phpipam
# 验证:48 张表 + 默认 Admin 用户;默认口令 Admin/ipamadmin(首登强制改)

1.4DNS + Ingress + SealedSecret

# FreeIPA A 记录
kinit yiwei && ipa dnsrecord-add djch.one phpipam --a-rec=192.168.6.240
# ingress 复用 *.djch.one 通配证书 wildcard-djch-one-tls(class: traefik / websecure)
# 封 secret 进 git(kubeseal 版本对齐控制器 0.37.0)
kubectl -n infra-nt get secret phpipam-db -o yaml \
 | kubeseal --controller-namespace sealed-secrets --format yaml \
 > manifests/infra-nt/phpipam-db.sealed.yaml
kubectl -n infra-nt delete secret phpipam-db   # 删明文,让控制器重建为它所有
SealedSecret 接管坑:控制器不覆盖非它创建的明文 secret(报 already exists and is not managed)。删明文 secret 后,删掉再重建 SealedSecret 触发 reconcile,pod 不重启(env 已注入,值一致)。

2 phpIPAM SSO · Authentik SAML2

// IdP=Authentik(auth.djch.one)· SP=phpIPAM · JIT 自动建号

2.1Authentik 侧:Provider + Application + JIT 属性(ak shell,幂等)

kubectl -n authentik exec -i deploy/authentik-server -- ak shell < authentik-setup.py
# 建 SAML Provider phpipam:
#   ACS = https://phpipam.djch.one/index.php?page=saml2   (未开 prettyLinks,非 /saml2/)
#   issuer/audience = https://phpipam.djch.one · sp_binding=post · 签名走 self-signed
#   NameID = Username(email 当 NameID 时无邮箱账号会空 → getNameId() 失败)
# 建 3 个自定义 SAMLPropertyMapping(裸名,Authentik 默认是 URI 名不匹配):
#   display_name / email(必填,加兜底 user.username+@djch.one)/ is_admin(=user.is_superuser)
三连坑:①ACS 不匹配 Authentik 报 ACS URL ... doesn't match;②NameID 空 → Could not extract valid username;③JIT 缺属性 → Mandatory SAML JIT attribute missing: display_name。契约在容器 app/saml2/index.php:必填 display_name+email,选填 is_admin/groups/modules。

2.2phpIPAM 侧:插 SAML2 认证方法(存 usersAuthMethod 表)

# 从 Authentik provider 拉 IdP 证书,写 usersAuthMethod(见 phpipam-authmethod.sh,幂等)
INSERT INTO usersAuthMethod (type,params,protected,description)
VALUES ('SAML2', '{...idpissuer/idplogin/idpx509cert/jit:1/strict:0...}', 'No', 'Authentik SAML SSO');
# idplogin = https://auth.djch.one/application/saml/phpipam/sso/binding/redirect/
验证:/index.php?page=saml2 出现 SAML 入口 → 跳 Authentik → 回跳 JIT 建号。is_admin 随 Authentik 超管态定 phpIPAM 角色。管理仍有本地 Admin/ipamadmin 兜底。

3 Foreman External IPAM

// provisioning 建主机自动从 phpIPAM 取空闲 IP 并登记
坑:smart_proxy_ipam 官方 RPM 在 Foreman 3.18/Katello 4.20/Rocky9 仓里没有,foreman-installer 也不认 → 只能手动装 gem(非 RPM 管理,Foreman 升级后可能要重装)。dry-run 确认零依赖冲突后再装。

3.1Foreman 装插件

# 先 dry-run 确认只装叶子 gem(不动 foreman-proxy 核心 gem)
gem install smart_proxy_ipam --explain
gem install smart_proxy_ipam --no-document
echo "gem 'smart_proxy_ipam'" > /usr/share/foreman-proxy/bundler.d/smart_proxy_ipam.rb
systemctl restart foreman-proxy   # 起不来就删上面那行回滚

3.2phpIPAM 侧:服务账号 + API app(认证机制关键)

认证机制:smart_proxy_ipam 走 POST url/api/{user}/user/ + basic auth 取 token → phpIPAM app_id 必须 = 配置里的 user,且要有同名 phpIPAM 用户。app_security 必须 ssl_token(enum 里的 user 会报 Invalid app security,API 只认 ssl_token/ssl_code/crypt/none)。
# phpIPAM DB:开 API + 建 local 用户 foreman(Administrator)+ API app foreman(ssl_token,rw)
UPDATE settings SET api=1;
INSERT INTO users (username,authMethod,password,role,...) VALUES ('foreman',1,'<bcrypt>','Administrator',...);
INSERT INTO api (app_id,app_permissions,app_security,app_comment)
VALUES ('foreman',2,'ssl_token','Foreman smart_proxy_ipam');
# 建子网(经 API,顺带验证认证):POST /api/foreman/subnets/ subnet=192.168.6.0 mask=24 sectionId=1

3.3Foreman 侧:proxy 配置 + 挂子网

# /etc/foreman-proxy/settings.d/externalipam.yml
:enabled: true
:use_provider: externalipam_phpipam
# /etc/foreman-proxy/settings.d/externalipam_phpipam.yml (640 root:foreman-proxy)
:url: 'https://phpipam.djch.one'
:user: 'foreman'
:password: '<foreman 用户密码>'
# restart 后刷新 feature + 挂子网
hammer proxy refresh-features --id 1        # 出现 External IPAM
hammer subnet update --id 1 --ipam "External IPAM" --externalipam-id 1

3.4端到端验证

# 用 Foreman 服务器的 client_cert 调 proxy(proxy 自己的 ssl_cert 当客户端证书会被拒)
curl --cert /etc/foreman/client_cert.pem --key /etc/foreman/client_key.pem \
  --cacert /etc/foreman/proxy_ca.pem \
  https://foreman.djch.one:9090/ipam/subnet/192.168.6.0/24/next_ip
# → {"data":"192.168.6.1"}  ✅

4 LibreNMS 部署

// 自包含 MariaDB + Redis + dispatcher(poller)· SNMP 自动发现

4.1清单要点

组件说明
redisredis:7.2-alpine,缓存/会话/dispatcher 锁
mariadb:10带 --character-set-server=utf8mb4 --lower-case-table-names=0,Longhorn 10Gi
librenms(双容器)web:8000 + dispatcher(SIDECAR_DISPATCHER=1),共享 /data PVC,strategy Recreate
kubectl create namespace librenms
kubectl -n librenms create secret generic librenms-db \
  --from-literal=mysql-root-password=$(openssl rand -hex 16) \
  --from-literal=mysql-password=$(openssl rand -hex 16)
kubectl apply -f manifests/librenms/librenms.yaml
两个必修坑:①containerd 下 web 容器 nginx/php-fpm: /proc/self/fd/2 Permission denied(FPM init failed)→ web 容器加 tty: true;②反代下 302 发 http → configmap 设 APP_TRUSTED_PROXIES: "*"(trustedproxy.php 默认只信 127.0.0.1)。

4.2建 admin + 自动发现

lnms user:add -r admin -p '<pw>' -e you@djch.one -l Admin admin
lnms config:set nets.0 192.168.6.0/24
lnms config:set snmp.community '["public"]'
lnms config:set discovery_by_ip true
snmp-scan 定时扫子网,设备侧要开 SNMP 且 community=public 才会被纳入。入口 https://librenms.djch.one。

5 IP 台账导入 · RouterOS ARP → phpIPAM

// 让 phpIPAM 知道现网已用 IP,Foreman 才不会分配撞车
ping 扫会漏掉防火墙挡 ICMP 的主机。用网关 RouterOS 的 ARP 表(L2,含所有活动设备+MAC)最准,再 FreeIPA 反向 DNS 补主机名。
# 1. 拉网关 ARP(完整条目)
ssh kissu8@192.168.6.254 '/ip arp print' | grep complete
# 2. 逐 IP 反向解析(FreeIPA .181)
dig +short -x 192.168.6.X @192.168.6.181
# 3. 拼 INSERT 灌 phpIPAM(ip_addr = ip2long,192.168.6.0=3232237056)
INSERT INTO ipaddresses (subnetId,ip_addr,is_gateway,description,hostname,mac,state)
VALUES (7, 3232237056+<octet>, 0, '<short>', '<fqdn>', '<mac>', 2);
# .254 标 is_gateway=1
本次导入 69 个在用 IP(68 ARP 活动 + 网关),26 个带反向 DNS 主机名,全部含 MAC。之后 next_ip 自动跳过它们。

6 PowerDNS 对接 待接

// phpIPAM 分配/改 IP → 自动在 PowerDNS 建 A/PTR 记录
PowerDNS 由另一个 agent 部署中。就绪后需其 API 地址(如 http://pdns:8081)+ API Key + 服务器 id。phpIPAM 原生支持,配置存 DB(Administration → PowerDNS),不进 GitOps。

phpIPAM 侧配置

  • Administration → PowerDNS:填 API URL / API Key / server(默认 localhost)
  • 或 DB 直连模式(pdns MySQL 后端库)
  • 开 Reverse DNS:子网勾 "Create PTR"

联动效果

  • phpIPAM 给主机分配 IP + hostname → 推 A 记录到正向区
  • 自动建 PTR 到 6.168.192.in-addr.arpa
  • Foreman 经 phpIPAM 取 IP 时,DNS 同步落地
# 需从 PowerDNS agent 拿到:
PDNS_API_URL=http://<pdns-host>:8081/api/v1
PDNS_API_KEY=<key>
# zones:djch.one(正向)· 6.168.192.in-addr.arpa(反向)
