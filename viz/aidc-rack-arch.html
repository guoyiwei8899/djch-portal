数据模型

// backend/app/models.py · 机柜与服务器经 rack_name 软关联(denormalized),PDU/出口经 FK 硬关联

Rack

  • name 唯一 · site_id → sites(SET NULL)
  • u_height 机柜 U 高(默认 42)= 渲染 totalU
  • 占位服务器靠 Server.rack_name == rack.name 反查,非外键

Server(机柜相关字段)

  • rack_name 所属机柜名(冗余字段)
  • rack_unit 起始 U 位(1-based,自底向上)
  • rack_height 占用 U 数(默认 1)→ 卡片高度
  • type HPC / Lustre / ALL → 卡片渐变配色

RackPDU

  • rack_id → racks(CASCADE) · side left/right
  • api_type(默认 unknown,apc 才支持电源控制)
  • management_ip + username/password 走 PDU API
  • outlet_count 出口数(默认 24)

RackPDUOutlet

  • pdu_id → rack_pdus(CASCADE) · outlet_index
  • power_state unknown / on / off / error
  • server_id → servers(SET NULL):出口↔服务器映射
  • detail 聚合时把出口按 server_id 归到占位卡的 outlet_ids

状态灯与健康判定

// integrations/prometheus.py(主)+ zabbix.py(回退)· ipmi_up / sel_count 决定灯色 · 仅 lamp_state==unknown 才回退

Prometheus(主源)

  • get_hosts_health(servers) 按 BMC IP 查
  • PromQL:ipmi_up{instance} + ipmi_sel_logs_count{instance}
  • monitor_source=prometheus,超时 15s
  • 失败则整端点回退 Zabbix,detail 带 monitor_error

灯色判定

  • ipmi_up==false → offline 红
  • sel_count>0 → sel 琥珀(active_problems)
  • ipmi_up==true → online 绿
  • 查无数据 → unknown 灰(触发回退)

Zabbix(回退 + 指标)

  • 回退健康:ipmi_available 1/2 → online/offline + problem 查 SEL
  • 指标 get_hosts_metrics:cpu_temp / inlet_temp / power_total
  • item key:hw.cpu.temp · hw.inlet.temp · hw.power.total
  • 指标喂悬停卡,与健康分两条独立查询

占用率统计

  • used_u = Σ server.rack_height
  • free_u = max(total_u - used_u, 0)
  • occupancy_percent = round(used_u/total_u×100)
  • 列表页利用率条:≥80% 红 / ≥60% 琥珀
